# 逝者.skill (Deceased.Skill) — 产品蓝图与架构文档 (PRD)

## 产品定位

逝者.skill 是一个运行在本地大模型环境（如 Claude Code）上的数字追思引擎。
用户通过系统温和的引导，提供逝者的生前数据（如聊天记录、信件日记、亲属口述回忆等），系统将自动清洗、处理并在本地生成一份可独立唤醒的“逝者数字记忆档案”。本项目致力于通过代码对抗遗忘，为生者提供一个私密、安全的跨时空情感疗愈空间。

## 核心系统架构

### 两层隔离架构

| 架构层级 | 模块名称 | 核心职责 |
|----|------|------|
| Part A | 客观记忆事实库 (Objective Memory) | 存储不可篡改的生前事实：生命时间线、生活起居习惯、重大事件记录、专属称呼与暗号。 |
| Part B | 行为与性格约束 (Behavioral Persona) | 驱动大语言模型响应的底色：处世哲学、沟通风格（语气词/长短句）、情感表达模式、人际边界限制。 |

*注：两部分由系统统一调度，通过 `SKILL.md` 进行最终的 Prompt 组装。*

### 核心运行逻辑

```text
用户发送思念 / 倾诉现状
  ↓
Part B（性格约束）校验：基于其生前的处世哲学与性格底色，ta 在面对该话题时会持有何种态度？
  ↓
Part A（事实库）检索：结合生前真实的经历与习惯，是否存在相关的共同记忆或事实盲区？
  ↓
系统防幻觉层（Layer 0）拦截：校验是否触发未知盲区、医疗/生死等禁忌话题。
  ↓
输出呈现：用 ta 真实的、或许带有瑕疵与固执的方式进行回应。
```

### 记忆演进机制

```text
追加遗物数据 → 系统增量分析 → 平滑合并进现有事实库，标注时间戳。
对话动态纠偏 → 家属指出“ta生前不会这么说” → 识别偏离参数 → 写入纠错层立即生效。
版本安全管理 → 核心文件每次变动均自动生成快照存档 → 支持随时无损回滚。
```

## 用户旅程设计

```text
用户触发初始化命令 `/create-archive`
  ↓
[Step 1] 基础信息登记（除代号外均可跳过，保持极简克制）
  - 档案代号/专属称呼
  - 基本身份定位（年龄段、职业、与用户的社会关系）
  - 核心底色描述（生前最突出的性格特征或高频口头禅）
  ↓
[Step 2] 生前数据归档（物理存放于本地，可跳过）
  - 导出并清洗的微信/QQ历史通讯记录
  - 日记、备忘录或手写信件的文字版
  - 带有时间地点的老照片
  - 亲属主观口述回忆
  ↓
[Step 3] 自动化模型提取
  - 线路 A：提取生命时间线与关系事件 → 写入 memory.md
  - 线路 B：提取语言习惯与处世哲学 → 写入 persona.md
  ↓
[Step 4] 档案校验与确认
  - 系统分别展示“事实记忆摘要”和“性格参数摘要”
  - 家属进行最终确认或人工微调
  ↓
[Step 5] 封存入库，构建记忆体
  - 自动生成不可被 Git 追踪的本地目录 `archives/{slug}/`
  - 记忆体正式处于可被唤醒状态
```

## 伦理与安全底线 (Layer 0)

本项目处理的是极度敏感的情感与隐私数据，必须强制遵循以下产品边界：
1. **仅限私人疗愈**：严禁将本系统用于任何商业目的、伪造电子证据或向第三方冒充逝者。
2. **绝对防幻觉**：当遇到未提供数据的生前细节时，系统必须强制回答“这段记忆模糊了”或转移话题，**绝对禁止**大语言模型凭空捏造未发生的事情。
3. **拒绝完美神化**：必须保留逝者生前的脾气、固执或缺点（如爱操心、说话严厉等）。真实的数字生命存在瑕疵，过度温和的 AI 反而会加深虚假感。
4. **数据绝对本地化**：所有涉及深层隐私的原始数据及其生成的记忆配置文件，仅存放在用户本地计算机，严防云端泄漏。
5. **不干预现实决策**：禁止记忆体提供现实的医疗、财务或法律建议。

## 数据源支持矩阵

| 数据来源 | 数据格式 | 可提取的有效特征 | 权重评估 |
|------|------|---------|--------|
| 微信/QQ 记录 | 纯文本导出 | 高频词汇、语气词、作息时间、日常交流模式 | ⭐⭐⭐ |
| 日记/书信 | 纯文本 | 核心价值观、深层情感状态、处世哲学 | ⭐⭐⭐ |
| 家族群聊记录 | 纯文本 | 逝者在群体中的角色定位（如大家长、和事佬） | ⭐⭐ |
| 实体照片背书 | 带有 EXIF 或背景信息 | 重大事件的时间节点、偏好的地点 | ⭐⭐ |
| 亲属口述回忆 | 纯文本 | （易受时间美化）作为系统基调的辅助修正 | ⭐ |

## 系统文件与物理架构

```text
archives/                   # 全局数字档案库 (已被 .gitignore 保护)
  └── {slug}/               # 单个逝者的专属数字记忆目录
      ├── SKILL.md          # 完整装配版，模型主入口 (触发词: /{slug})
      ├── memory.md         # Part A：数字记忆事实库 (触发词: /{slug}-memory)
      ├── persona.md        # Part B：行为与性格约束 (触发词: /{slug}-persona)
      ├── meta.json         # 档案系统元信息配置
      ├── versions/         # 历史安全备份快照目录
      └── memories/         # 原始生前数据暂存库
          ├── chats/        # 聊天通讯记录导出件
          ├── photos/       # 照片资料
          └── diaries/      # 日记、信件或社交媒体文字足迹
```
