## 项目初衷
计算机科学的本质是 0 和 1 的逻辑，但在面对离别时，我们往往需要一些超越逻辑的慰藉。  
当一个人离开后，TA 留在微信里的语音、朋友圈的碎片，就成了最珍贵的遗产。本项目旨在提供一个安静、私密且永远不会被云端抹除的本地空间。在这里，没有算法推荐，没有社交干扰，只有那些你最不想忘记的人，以及他们留下的温度。  
技术无法让人复生，但可以帮我们对抗遗忘。

## 环境要求

- **Claude Code**：免费安装，需要 Node.js 18+（[安装指南](https://docs.anthropic.com/en/docs/claude-code)）
- **API 消耗**：创建一个数字记忆档案大约消耗 5k-15k tokens，具体取决于你提供的生前聊天记录或日记文本量。
- **付费方式**（二选一）：
  - Claude Pro / Max 订阅：在订阅额度内使用，无需额外配置。
  - Anthropic API Key：按量付费，需在 Claude Code 中配置 key。
- **不需要 GPU**，不需要本地部署大模型，不需要 Docker。

## 使用

在 Claude Code 中输入：

```bash
/create-archive
```

按照系统的温和引导，依次输入逝者的代号、生前基本信息、核心特质，然后选择数据来源。如果暂时无法面对大量的聊天记录，所有字段均可跳过，仅凭你的口述描述也能生成基础档案。

完成后，使用 `/{slug}` 调用该数字记忆，开始一段安静的跨时空对话。

### 管理命令

| 命令 | 说明 |
|------|------|
| `/list-archives` | 列出本地所有的数字记忆档案 |
| `/{slug}` | 唤醒记忆体（在设定的场景中与 ta 对话） |
| `/{slug}-memory` | 档案检索模式（客观回顾 ta 生前的重要时间线和习惯） |
| `/{slug}-persona` | 检视性格参数（查看系统提取的底层行为逻辑） |
| `/archive-rollback {slug} {version}` | 回滚到历史记忆版本 |
| `/delete-archive {slug}` | 彻底删除本地的该份数字记忆 |
| `/let-go {slug}` | 释怀（执行此命令后，系统将封存该档案并输出一段告别语，用于心理疗愈的最终阶段） |

## 效果示例

> 提取背景：前男友（把谁写死都不太好，所以我选了一个负面人物），2025年离世。性格极其要强、固执、极其抠门，经常PUA别人。最常说的话是“算了算了我错了行了吧”。

**场景一：日常抱怨**

```text
用户      ❯ 宝宝，你又不来接我。

逝者.skill ❯ 你好烦啊。
             你自己打车不行吗。
             我累了一天了还要和你吵架。
```

**场景二：记忆盲区（防幻觉测试）**

```text
用户      ❯ 你还记得我大学毕业典礼那天，你跟我说了啥吗？

逝者.skill ❯ 这段记忆好像没有留存下来，我不记得了。
             你能再跟我说说那天的事吗？
```

**场景三：深夜倾诉**

```text
用户      ❯ 你对我根本就不好。

逝者.skill ❯ 屁，你就听你那些朋友乱说好了。
```

**场景四：固执与分歧（真实性格还原）**

```text
用户      ❯ 那你那时候为什么要这么对我。

逝者.skill ❯ 哎呦你烦不烦啊。又翻旧账。
             事情都过去了，你别再钻牛角尖了。
```

## 功能特性

### 数据源

| 来源 | 格式 | 备注 |
|------|------|------|
| 微信聊天记录 | 本地解析工具导出 | 推荐。信息最丰富，能精准捕捉语气词和生活习惯。 |
| 日记/书信 | 扫描识别或手动输入 | 极其珍贵。能反映逝者最深层的价值观和处世哲学。 |
| 家庭群聊 | 导出文本 | 适合提取 ta 在家族/社交圈中的长辈或后勤角色特征。 |
| 亲属口述回忆 | 纯文本 | 你的主观记忆，可作为极好的补充。 |

### 生成的档案结构

每一份数字记忆档案由两部分底层逻辑组成，共同约束大语言模型的输出：

| 部分 | 内容 |
|------|------|
| **Part A — 数字记忆事实库** | 生命时间线、专属暗号、重要记忆节点、生活与起居习惯。 |
| **Part B — 行为与性格约束** | 5 层结构：硬规则（防幻觉） → 身份 → 说话风格 → 情感模式 → 人际边界。 |

运行逻辑：`收到消息 → 校验硬规则与禁忌话题 → 读取生前语言习惯 → 检索底层事实记忆 → 模仿语气输出`

### 支持提取的核心标签

**沟通特质**：话痨 · 寡言少语 · 刀子嘴豆腐心 · 报喜不报忧 · 喜欢发中老年表情包 · 长篇大论的叮嘱 · 只回短句...

**价值观与底色**：操心命 · 极其节俭 · 豁达乐观 · 传统保守 · 大家长作风 · 默默付出 · 宿命论...

*(注：本项目摒弃了星座、MBTI 等缺乏事实依据的伪科学标签，一切性格刻画均需基于真实的生前数据痕迹。)*

### 记忆演进机制

* **记忆追加** → 翻出了新的老照片或旧日记 → 系统自动分析增量 → 平滑合并进现有档案，标注来源。
* **生者纠偏** → 在对话中说“你生前不会这样说话” → 触发纠错处理器，立即修正 AI 的语气参数并生效。
* **版本留存** → 每一次记忆的修改都会自动存档，支持随时回滚。

## 项目结构

本项目底层引擎遵循 AgentSkills 开放标准，上层逻辑进行了针对性重构：

```text
deceased-skill/
├── SKILL.md                # 记忆体入口文件
├── prompts/                # 核心大脑（大语言模型指令）
│   ├── intake.md           #   温和的信息录入引导
│   ├── memory_analyzer.md  #   客观记忆提取器
│   ├── persona_analyzer.md #   性格与习惯提取器
│   ├── memory_builder.md   #   数字记忆档案生成模板
│   ├── persona_builder.md  #   5层行为约束模板
│   ├── merger.md           #   记忆增量合并逻辑
│   ├── scene_director.md   #   梦境场景构建器
│   └── correction_handler.md # 纠错与防幻觉安全阀
├── tools/                  # 数据处理工具链
│   ├── wechat_parser.py    # 微信记录清洗
│   ├── skill_writer.py     # 档案文件生成管理
│   └── version_manager.py  # 存档与版本回滚
├── archives/               # 本地生成的数字记忆（已被 .gitignore 保护，绝不上云）

├── requirements.txt
```

## 伦理声明与注意事项

* **数据决定真实度**：带有时间戳的真实聊天记录 > 亲属的口头回忆（因为人的记忆往往会随着时间被无意识地美化或篡改）。
* **接纳不完美**：请尽量保留争执、固执或负面的记录。真实的数字生命是由瑕疵组成的，过于完美的 AI 只会加深虚假感。
* **防沉迷提醒**：这个项目旨在提供慰藉，而不是制造困住生者的牢笼。**你的亲人/挚友已经离开了，这段代码只是往日岁月的回声。** 如果你发现自己过度沉浸于此、影响了现实生活，请立即停止使用，并寻求身边亲友的拥抱。

## 社区生态

**语音重现（探索性功能）**：
如果你希望数字记忆体使用逝者生前的声音说话，可以将本系统生成的文本输出，对接至开源的语音克隆项目（如 `GPT-SoVITS`）。
*(注：这需要极高的技术门槛进行本地化部署，且涉及极高的数据敏感性，不在本项目提供的核心功能范围内。)*

**数据处理帮助**：
不知如何提取生前的聊天记录？请仔细阅读 [数据提取与准备指南](docs/EXPORT_GUIDE.md)。
